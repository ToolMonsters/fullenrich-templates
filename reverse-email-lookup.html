Turn email leads into full B2B profiles with FullEnrich

Got a list of emails from ads, forms, or Gmail? Use FullEnrich's API to reverse-lookup each email and get the person's full name, job title, company, and phone number.

What this does: For each email in your list, FullEnrich reverse-looks up the person and returns their full name, job title, company name, and phone number. You get a complete B2B profile from just an email address.
API key required: This template uses FullEnrich's API directly via HTTP requests. You'll need your FullEnrich API key (find it in your FullEnrich dashboard under Settings → API).
1
Create a scenario and add your email source trigger
Go to make.com and click Create a new scenario. Add a trigger module for your email source. This could be:
  • Google Sheets — "Watch Rows" on a sheet of email addresses
  • Gmail — "Watch Emails" to process incoming emails
  • Any other trigger — webhooks, ad lead forms, CSV imports, etc.
The key field you need from this trigger is the email address.
Screenshot: Trigger module setup (e.g. Google Sheets)
2
Add an HTTP Request module to call FullEnrich
Click the + to add a new module. Go to HTTP → Make a request. Configure it as follows:
  • URL: https://api.fullenrich.com/api/v1/contact/enrich
  • Method: POST
  • Headers:
    • Authorization = Bearer YOUR_API_KEY
    • Content-Type = application/json
  • Body type: Raw (JSON)
  • Body content:
{ "email": "{{1.email}}", "enrich_fields": "emails_and_phones" }
Screenshot: HTTP Request module configuration
3
Parse the response
Add a JSON → Parse JSON module after the HTTP Request. Connect the response body as input. This will extract the enriched fields, including:
  • first_name
  • last_name
  • current_job_title
  • current_company_name
  • most_probable_phone
Screenshot: Parse JSON module output
4
Write the enriched data back to your destination
Add a module for your destination (e.g. Google Sheets → Update a Row, Salesforce → Update Record, or any other CRM/database). Map the parsed fields:
  • Full Name → first_name + last_name
  • Job Title → current_job_title
  • Company → current_company_name
  • Phone → most_probable_phone
Screenshot: Destination module field mapping
5
Test and activate
Click Run once to test the full scenario. Confirm that the enriched data (name, title, company, phone) appears correctly in your destination. Then toggle the scenario ON and set the schedule.
Flexible email sources: You can use any email source as the trigger: Gmail, Google Sheets, ad lead forms, webhooks, etc. The HTTP Request module works the same regardless of where the email comes from.

Troubleshooting

ProblemSolution
HTTP Request returns 401 UnauthorizedDouble-check your API key in the Authorization header. It should be Bearer YOUR_API_KEY (with a space after "Bearer").
FullEnrich returns empty resultsThe email may not be in FullEnrich's database. Try with a known business email to verify your setup is correct.
Parse JSON failsMake sure you're connecting the HTTP Response body (not the full response object) to the Parse JSON module.
Destination module fails to updateCheck that you're mapping the correct row/record ID so the module knows which row to update.

Create a new workflow in n8n. Add a trigger node for your email source. This could be:

  • Gmail Trigger — process incoming emails
  • Webhook — receive emails from ad lead forms or other systems
  • Any other trigger — CSV file, CRM export, etc.

Screenshot: Trigger node setup (e.g. Google Sheets Trigger)
2
Add an HTTP Request node to call FullEnrich
Add an HTTP Request node after the trigger. Configure it as follows:
  • URL: https://api.fullenrich.com/api/v1/contact/enrich
  • Method: POST
  • Authentication: Header Auth
    • Header Name: Authorization
    • Header Value: Bearer YOUR_API_KEY
  • Body Content Type: JSON
  • Body:
{ "email": "{{ $json.email }}", "enrich_fields": "emails_and_phones" }
Screenshot: HTTP Request node configuration
3
Review the response
Run the workflow once to test. The HTTP Request node's output will contain the enriched data. You'll see fields like:
  • first_name
  • last_name
  • current_job_title
  • current_company_name
  • most_probable_phone
n8n automatically parses JSON responses, so you can reference these fields directly in subsequent nodes.
Screenshot: HTTP Request node output with enriched fields

Write the enriched data back to your destination
Add a node for your destination. For example:
  • Google Sheets → Update Row
  • Salesforce → Update Lead
  • HubSpot → Update Contact
Map the enriched fields:
  • Full Name → {{ $json.first_name }} + {{ $json.last_name }}
  • Job Title → {{ $json.current_job_title }}
  • Company → {{ $json.current_company_name }}

Test and activate

Troubleshooting

ProblemSolution
HTTP Request returns 401 UnauthorizedDouble-check your API key. In n8n's Header Auth credentials, the name should be Authorization and the value should be Bearer YOUR_API_KEY (with a space after "Bearer").
FullEnrich returns empty resultsThe email may not be in FullEnrich's database. Try with a known business email to verify your setup is correct.
