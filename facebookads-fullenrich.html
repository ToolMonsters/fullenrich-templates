Enrich Facebook Ads leads with FullEnrich

Automatically enrich every new Facebook Lead Ad submission with verified B2B emails, phone numbers, and LinkedIn profiles using FullEnrich.

What this Zap does: When a new lead is captured by a Facebook Lead Ad, FullEnrich's Reverse Email Lookup API identifies the person behind the personal email and returns their full B2B profile (work email, phone, company, job title, LinkedIn), then sends the enriched data to Google Sheets.
Why Reverse Email Lookup? Facebook Lead Ads typically capture personal emails (Gmail, Yahoo, etc.). The Reverse Email Lookup endpoint (POST /contact/reverse/email/bulk) takes a personal email and returns the full Person & Company profile — work email, direct phone, LinkedIn, company name, and job title. 1 credit is used per successful match.
1
Create a new Zap — Facebook Lead Ads trigger
Go to zapier.com and click Create Zap. For the trigger, search for Facebook Lead Ads and select the event New Lead. Connect your Facebook account. Select the Page and Form you want to capture leads from. Click Test trigger and make sure you can see the lead's Email field.
2
Add Webhooks by Zapier — call the Reverse Email Lookup API
Add an action step: Webhooks by Zapier → Custom Request. Configure:
  • Method: POST
  • URL: https://api.fullenrich.com/contact/reverse/email/bulk
  • Headers: Authorization: Bearer YOUR_API_KEY and Content-Type: application/json
  • Body: {"name": "Facebook Ads Reverse Lookup", "data": [{"email": "{{lead email}}"}]} — map the lead's personal email from Step 1.
The API returns an enrichment_id. FullEnrich processes the lookup asynchronously.
3
Add a Delay step
Add Delay by Zapier → Delay For and set it to 1 minute. This gives FullEnrich time to complete the reverse lookup before you fetch the results.
4
Fetch the enrichment results
Add another Webhooks by Zapier → Custom Request:
  • Method: GET
  • URL: https://api.fullenrich.com/contact/reverse/email/bulk/{{enrichment_id}} — map the enrichment_id from Step 2's response.
  • Headers: Authorization: Bearer YOUR_API_KEY
The response contains the enriched Person & Company data for each email.
5
Send enriched data to Google Sheets
Add Google Sheets → Create Spreadsheet Row. Map the columns from the enrichment result:
  • Personal Email → Lead's Email (from Facebook)
  • Work Email → data[0].contact.most_probable_email
  • Phone → data[0].contact.most_probable_phone
  • LinkedIn → data[0].contact.social_medias[0].url
  • First Name, Last Name, Company, Job Title → from the Person profile in the response
6
Test and publish
Test the full Zap, then click Publish. Every new Facebook Lead Ad submission will be automatically reverse-looked-up and enriched with B2B contact data.

Troubleshooting

ProblemSolution
Enrichment status is not FINISHEDIncrease the Delay step to 2 minutes. Some lookups take longer depending on the email provider.
FullEnrich returns no matchNot all personal emails can be matched to a B2B profile. This is normal — 1 credit is only used when a match is found.
401 UnauthorizedCheck that your API key is correct in the Authorization header: Bearer YOUR_API_KEY (with the space after Bearer).
Zap doesn't triggerEnsure the trigger is set to New Lead and that you selected the correct Facebook Page and Lead Form. Verify your Facebook account permissions.

Toggle this scenario ON.
2
Create Scenario A — Facebook Lead Ads trigger + API call
Create another scenario. Add the first module: Facebook Lead Ads → Watch Leads. Connect your Facebook account and select the Page and Form to monitor. This is an instant trigger that fires when a new lead is submitted.
3
Add HTTP module — Reverse Email Lookup API call
Add the module HTTP → Make a request. Configure:
  • URL: https://api.fullenrich.com/contact/reverse/email/bulk
  • Method: POST
  • Headers: Authorization: Bearer YOUR_API_KEY and Content-Type: application/json
  • Body type: Raw → JSON
  • Request content:

Test and activate
Run once on Scenario A to send a test lookup. After a few seconds, Scenario B should receive the enriched results via webhook and add a row to Google Sheets. Once confirmed, toggle both scenarios ON.

What this workflow does: When a new lead is captured by a Facebook Lead Ad, an HTTP Request calls FullEnrich's Reverse Email Lookup API with the lead's personal email. When the lookup is complete, FullEnrich sends the enriched B2B profile (work email, phone, LinkedIn, company, job title) to a webhook, which then writes the data to Google Sheets.
Why two flows? The Reverse Email Lookup API (POST /contact/reverse/email/bulk) is asynchronous. You send the email, and FullEnrich delivers the results to your webhook when ready. The top flow sends the request; the bottom flow receives and processes the results.
1
Create the top flow — Facebook Lead Ads trigger
Create a new workflow in n8n. Add a Facebook Lead Ads Trigger node (or use a Webhook node that Facebook posts to). Connect your Facebook account and select the Page and Form you want to monitor for new leads.
2
Add HTTP Request — Reverse Email Lookup
After the trigger, add an HTTP Request node:
  • Method: POST
  • URL: https://api.fullenrich.com/contact/reverse/email/bulk
  • Authentication: Generic Credential Type → Header Auth → Name: Authorization, Value: Bearer YOUR_API_KEY
  • Body (JSON):
    {"name": "Facebook Ads Reverse Lookup", "data": [{"email": "{{ $json.email }}"}], "webhook_url": "YOUR_WEBHOOK_PRODUCTION_URL"}
Map {{ $json.email }} to the lead's personal email from the trigger. The webhook_url points to the Webhook node in Step 3.
3
Create the bottom flow — Webhook receiver
Add a Webhook node (this starts a separate flow). Set:
  • HTTP Method: POST
  • Path: fb-lead-reverse-lookup
Copy the production URL and paste it as the webhook_url in Step 2's HTTP Request body. When FullEnrich finishes the reverse lookup, it sends the results here.
4
Send enriched data to Google Sheets
After the Webhook node, add a Google Sheets node set to Append Row. Map the fields from the webhook payload:
  • Work Email → {{ $json.body.data[0].contact.most_probable_email }}
  • Phone → {{ $json.body.data[0].contact.most_probable_phone }}
  • LinkedIn → {{ $json.body.data[0].contact.social_medias[0].url }}
  • First Name, Last Name, Company, Job Title → from the Person & Company profile in data[0]
5
Test and activate
Click Test workflow, then submit a test lead via your Facebook Lead Ad form. The HTTP Request sends the personal email to FullEnrich, and after a few seconds the webhook receives the enriched B2B profile. Check your Google Sheet for the new row. Toggle the workflow to Active and make sure the webhook uses the production URL.

Troubleshooting

ProblemSolution
Webhook never receives dataMake sure you're using the production URL (not webhook-test) and the workflow is Active.
401 error on HTTP RequestCheck the Authorization header: Bearer YOUR_API_KEY. Make sure the API key is valid.
No match found for emailNot all personal emails can be resolved to a B2B profile. 1 credit is only used when a match is found.
Google Sheets row incompleteVerify the field paths in the Google Sheets mapping. The webhook payload structure is body.data[0].contact.*.
Facebook trigger not firingVerify your Facebook app permissions and page/form selection. Submit a test lead from Facebook Ads Manager to confirm.
