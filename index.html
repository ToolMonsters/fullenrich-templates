Add the FullEnrich Enrichment

This step sends the contact's details to FullEnrich, which will find their verified mobile phone number.

  1. Click the + button below the HubSpot step to add a new action
  2. Search for FullEnrich and select it
  3. Choose the action: Enrich Contact

    • Company Name → select Company Name from HubSpot
    • LinkedIn URL → select LinkedIn URL from HubSpot (if available)
  6. Click Test step — FullEnrich will return the enrichment result, including the phone number

How this workflow is different: Unlike Zapier and Make, n8n uses an async (two-flow) pattern. FullEnrich takes a few minutes to find phone numbers, so the workflow is split in two: one flow sends the request, and a second flow receives the result via a webhook. Don't worry — it all happens automatically once set up!
1

Import the Workflow

First, you need to load the pre-built workflow into your n8n instance. This saves you from building everything from scratch — all the nodes and connections are already configured.

  1. Open your n8n instance and go to the Workflows page
  2. Click the ⋯ menu (top right) → select Import from File
  3. Upload the workflow JSON file we provide
  4. You'll see two separate flows on the canvas:
    • Top flow (3 nodes): HubSpot Trigger → FullEnrich — this sends new contacts for enrichment
    • Bottom flow (2 nodes): Webhook → HubSpot Update — this receives results and updates the contact
Screenshot 1 — The full workflow canvas
Show the entire canvas after import — both flows visible with sticky notes. The top flow has "Get recently created/updated contacts" → "Start FullEnrich", the bottom flow has "FullEnrich Result Webhook" → "Create or update a contact".
2

Connect HubSpot (Trigger Node)

The top flow starts with HubSpot detecting new contacts. You need to connect your HubSpot account so n8n can watch for new contacts automatically.

  1. Double-click the "Get recently created/updated contacts" node (first node in the top flow)
  2. Next to Credential to connect with, click the dropdown → Create New Credential
  3. Select OAuth2 and sign in to your HubSpot account
  4. Once connected, click Back to canvas
Screenshot 2 — HubSpot trigger node
Show the "Get recently created/updated contacts" node open, with the credential dropdown visible and a HubSpot account connected.
3

Connect FullEnrich

This is the node that sends each new contact to FullEnrich for enrichment. It automatically passes the contact's name, company, and domain. It also sends a webhook URL so FullEnrich knows where to send the results back.

  1. Double-click the "Start FullEnrich" node (second node in the top flow)
  2. Next to Credential to connect with, click the dropdown → Create New Credential
  3. Paste your FullEnrich API key (find it in your FullEnrich dashboard under Settings → API)
  4. Click Save

The fields are already mapped for you:

  • First Name, Last Name, Company Name, Company Domain — pulled from the HubSpot contact
  • LinkedIn URL — pulled from the contact if available (helps get better results)
  • Webhook URL — automatically set to the webhook in the bottom flow
  • Custom Fields — passes the HubSpot contact ID so the result can be matched back
Screenshot 3 — Key screenshot
Show the "Start FullEnrich" node open, scrolled to show all the mapped fields: firstName, lastName, companyName, companyDomain, linkedinUrl, webhookUrl, and customFields with the contact ID.
4

Connect HubSpot (Update Node)

The bottom flow ends with updating the contact back in HubSpot. This node writes the phone number that FullEnrich found. You need to connect your HubSpot account here too (you can reuse the same credential).

  1. Double-click the "Create or update a contact" node (second node in the bottom flow)
  2. Next to Credential to connect with, select the same HubSpot account you connected in Step 2
  3. Verify the field mapping is set:
    • Contact ID — comes from the webhook data (matches back to the original contact)
    • Mobile Phone Number — mapped to FullEnrich's most probable phone number
    • Email — mapped to FullEnrich's most probable email
  4. Click Back to canvas
Screenshot 4 — Key screenshot
Show the "Create or update a contact" node open, with the field mapping visible: Contact ID, Mobile Phone Number, and Email all mapped to the webhook data from FullEnrich.
5

Check the Webhook Node

The webhook is what allows FullEnrich to send the enrichment results back to n8n. It's already configured — you just need to make sure it's working. The webhook URL only becomes active when the workflow is turned on (next step).

  1. Double-click the "FullEnrich Result Webhook" node (first node in the bottom flow)
  2. You'll see the Webhook URL — this is the address FullEnrich will call when enrichment is done
  3. Make sure the HTTP method is set to POST
  4. No other changes needed — click Back to canvas
Important: The webhook URL has two versions: a test URL (with webhook-test) and a production URL (with webhook). The production URL only works when the workflow is Active. Make sure the URL in the FullEnrich node uses the production version.
Screenshot 5 — Webhook node
Show the "FullEnrich Result Webhook" node open, with the production webhook URL visible and HTTP method set to POST.
6

Activate the Workflow

Turning on the workflow does two things at once: it starts watching HubSpot for new contacts and it activates the webhook so FullEnrich can send results back. Both flows need to be active for the automation to work end-to-end.

  1. Click the Active toggle in the top-right corner of the canvas
  2. The toggle should turn green — this means both flows are now live
Screenshot 6 — Workflow activated
Show the workflow canvas with the Active toggle turned ON (green) in the top-right corner.
7

Test the Full Flow

Now let's make sure everything works. Since this is an async workflow, you'll see two separate executions in the log — one for each flow. The second one arrives a few minutes after the first.

  1. Go to HubSpot and create a new contact (or update an existing one)
  2. Go back to n8n → click Executions (left sidebar)
  3. Within a minute, you should see the first execution (top flow: HubSpot → FullEnrich)
  4. Wait 1–5 minutes. You'll see a second execution (bottom flow: Webhook → HubSpot Update)
  5. Go back to HubSpot and check the contact — the Mobile Phone Number field should now have a value
Screenshot 7 — Execution log
Show the n8n Executions page with two successful executions listed — the first one from the HubSpot trigger and the second one from the webhook, a few minutes apart.
Don't panic if the second execution takes a few minutes! FullEnrich searches multiple data sources to find the most accurate phone number. This is normal and expected.
Using Salesforce, Pipedrive, or another CRM? Simply replace the two HubSpot nodes with your CRM of choice. Use your CRM's trigger node in the top flow, and your CRM's update node in the bottom flow. The FullEnrich node, the webhook, and the async pattern all stay exactly the same.

Troubleshooting

Issue Solution
No phone number returned The contact may not have a findable mobile number. Check your FullEnrich dashboard for the enrichment result.
Phone field empty in HubSpot Make sure you're selecting Most Probable Phone Number from the FullEnrich step when mapping the field in HubSpot.
FullEnrich returns an error Check that your API key is valid and you have remaining credits in your FullEnrich account.
HubSpot connection expired Re-authenticate your HubSpot connection in your automation platform's settings.
Zap not triggering Make sure your Zap is published (not just saved). Also check that the HubSpot trigger is set to New Contact and not a different event.

Issue Solution
No professional profile found Not all personal emails can be matched. FullEnrich works best with real personal emails (Gmail, Outlook, Yahoo). Generic or disposable emails may not return results.
Only partial data returned Some fields may be unavailable depending on the person's online presence. The more data available publicly, the more FullEnrich can find.
FullEnrich returns an error Check that your API key is valid and you have remaining credits in your FullEnrich account.
Ad platform not sending leads Verify the trigger connection in your automation tool. For webhooks, make sure the URL is correct and the workflow/scenario is active.

Add FullEnrich Email Verification

This step sends each lead to FullEnrich for email enrichment and triple verification. FullEnrich finds the work email and checks if the inbox actually exists — not just if the domain is valid.

  1. Click + to add a new step
  2. Search for FullEnrich and select it
  3. Choose the action: Enrich Contact
  4. Connect your FullEnrich account using your API key
  5. Set Enrichment to Email Only — this tells FullEnrich to focus on finding and verifying emails
  6. Map the fields from your Google Sheet:
    • First Name → Column A
    • Last Name → Column B
    • Company Name → Column C
  7. Click Test step — you'll see the enrichment result including the email and its verification status
Tip: You can also map Domain (column D) and LinkedIn URL (column E) if your sheet has them. These are optional but help FullEnrich find the right email faster and more accurately.

4

Add a Filter

This is the step that protects your sender reputation. It checks the email verification status from FullEnrich and only lets leads with a confirmed, deliverable email continue to Instantly. Everything else is blocked.

  1. Click + to add a new step
  2. Search for Filter by Zapier (it's a built-in Zapier tool, no account needed)
  3. Set the condition:
    • Field: select Most Probable Email Status from the FullEnrich step
    • Condition: (Text) Exactly matches
    • Value: DELIVERABLE

When the filter blocks a lead, the Zap simply stops for that contact — no error, no notification. Catch-all and invalid emails are silently filtered out.

    • First Name → Column A from Google Sheets
    • Last Name → Column B from Google Sheets
    • Company Name → Column C from Google Sheets
  7. Click Test step to confirm the lead is added to your campaign

How this workflow is different: This n8n workflow uses an async (two-flow) pattern. FullEnrich takes a few minutes to verify emails, so the workflow is split in two: one flow sends the lead for verification, and a second flow receives the result and adds verified leads to your outreach tool.
1

Import the Workflow

We provide a pre-built workflow file so you don't have to create everything from scratch. All the nodes and connections are already configured for you.

  1. Open your n8n instance and go to the Workflows page
  2. Click the ⋯ menu (top right) → select Import from File
  3. Upload the workflow JSON file we provide
  4. You'll see two separate flows on the canvas:
    • Top flow (2 nodes): Google Sheets Trigger → FullEnrich — this sends new leads for email verification
    • Bottom flow (3 nodes): Webhook → Filter → Instantly — this receives results and only adds verified leads

Troubleshooting

Issue Solution
Bounce rate still above 1% Make sure the filter step is active and correctly checking the verification status. Only valid emails should pass through — exclude catch-all and unknown statuses.
Too many emails filtered out This is expected for low-quality lists. FullEnrich is protecting your sender reputation by removing risky addresses. Consider sourcing higher-quality prospects.
Lead not appearing in Instantly Check that your Instantly API key is correct and the campaign ID is valid. Verify the lead isn't already in the campaign (Instantly deduplicates by email).
FullEnrich returns an error Check that your API key is valid and you have remaining credits in your FullEnrich account.
Catch-all emails passing through Update your filter to explicitly exclude catch-all status in addition to invalid. Catch-all domains accept all emails but may still bounce.
