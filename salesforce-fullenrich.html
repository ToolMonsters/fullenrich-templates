Enrich new Salesforce leads with FullEnrich

Automatically enrich every new Salesforce lead with verified emails and phone numbers using FullEnrich, then update the lead record with the enriched data.

What this Zap does: When a new lead is created in Salesforce, FullEnrich finds their verified email and phone number, then updates the lead record automatically.
1
Create a new Zap
Go to zapier.com and click Create Zap. For the trigger, search for Salesforce and select the event New Record. Connect your Salesforce account and choose Lead as the object type.
Screenshot: Salesforce trigger setup
2
Test the trigger
Click Test trigger. Zapier will pull a recent lead from Salesforce. Make sure you can see the lead's First Name, Last Name, and Company fields in the test data.
Screenshot: Trigger test results
3
Add FullEnrich
Add a new action step and search for FullEnrich. Select the action Enrich Contact. Connect your FullEnrich account using your API key. Then map the fields:
  • First Name → Lead's First Name
  • Last Name → Lead's Last Name
  • Company Name → Lead's Company
  • Company Domain → Lead's Website (if available)
Screenshot: FullEnrich field mapping
4
Test the enrichment
Click Test step. FullEnrich will enrich the lead and return the results. You should see fields like Most Probable Email and Most Probable Phone in the output.
Screenshot: FullEnrich test results
5
Update the Salesforce lead
Add another action step: Salesforce → Update Record. Choose Lead as the object type. For Record, map the Lead ID from Step 1. Then map:
  • Email → Most Probable Email (from FullEnrich)
  • Phone → Most Probable Phone (from FullEnrich)
Screenshot: Salesforce update mapping
6
Turn on your Zap
Test the full Zap, then click Publish. Every new lead created in Salesforce will now be automatically enriched and updated with verified contact info.
Not using Salesforce? You can swap the Salesforce steps for any other CRM (HubSpot, Pipedrive, Zoho, etc.). The FullEnrich step stays the same.

Troubleshooting

ProblemSolution
FullEnrich returns no emailMake sure First Name, Last Name, and Company are mapped correctly. Adding the company domain improves match rates.
Salesforce update failsCheck that the Lead ID is correctly mapped from Step 1, and that your Salesforce user has edit permissions on the Lead object.
Zap doesn't triggerEnsure the trigger is set to New Record (not Updated Record) and that you're creating leads in the correct Salesforce org.
What this scenario does: When a new lead is created in Salesforce, FullEnrich finds their verified email and phone number, then updates the lead record automatically.

Screenshot: Salesforce Watch Records module
2
Add FullEnrich
Click the + to add a new module. Search for FullEnrich and select Enrich a Contact. Connect your FullEnrich API key. Map the fields:
  • First Name → Lead's FirstName
  • Last Name → Lead's LastName
  • Company Name → Lead's Company
  • Company Domain → Lead's Website (if available)
Screenshot: FullEnrich module mapping
3
Run the scenario once
Click Run once to test. Make sure a lead exists in Salesforce so the trigger picks it up. After the run, check the FullEnrich module output for Most Probable Email and Most Probable Phone.
Screenshot: FullEnrich output in Make
4
Update the Salesforce lead
Add a third module: Salesforce → Update a Record. Set the record type to Lead and map the Record ID from the first module. Then map:
  • Email → Most Probable Email (from FullEnrich)
  • Phone → Most Probable Phone (from FullEnrich)
Screenshot: Salesforce update module
5
Test the full scenario

6
Activate the scenario

Not using Salesforce? You can replace the Salesforce modules with any other CRM (HubSpot, Pipedrive, Zoho, etc.). The FullEnrich module stays the same.

What this workflow does: When a new lead is created in Salesforce, FullEnrich finds their verified email and phone number, then updates the lead record. Because FullEnrich works asynchronously in n8n, this workflow uses two flows: one to send the enrichment request, and one to receive the results via webhook.
Why two flows? FullEnrich enrichment takes a few seconds. Instead of waiting, FullEnrich sends the results to a webhook when ready. The top flow sends the request; the bottom flow receives and processes the results.
1
Create the top flow: Salesforce trigger
Create a new workflow in n8n. Add a Salesforce Trigger node and set it to watch for new Leads. Connect your Salesforce account.
Screenshot: Salesforce Trigger node
2
Add the FullEnrich node
Add a FullEnrich node after the Salesforce trigger. Connect your API key and map:
  • First Name → {{ $json.FirstName }}
  • Last Name → {{ $json.LastName }}
  • Company Name → {{ $json.Company }}
  • Company Domain → {{ $json.Website }}
Set the Webhook URL to the webhook node's production URL (from Step 3). Also add Custom Fields to pass through the Salesforce Lead ID:
  • Name: leadId — Value: {{ $json.Id }}
Screenshot: FullEnrich node with custom fields
3
Create the bottom flow: Webhook
Add a Webhook node (this starts a separate flow). Set the HTTP method to POST and give it a path like salesforce-enrich. Copy the production URL (not the test URL) and paste it in the FullEnrich node's Webhook URL field from Step 2.
Screenshot: Webhook node setup
4
Update the Salesforce lead
After the Webhook node, add a Salesforce node set to Update a Lead. Map:
  • Lead ID → {{ $json.body.datas[0].custom_fields.leadId }}
  • Email → {{ $json.body.datas[0].contact.most_probable_email.email }}
  • Phone → {{ $json.body.datas[0].contact.most_probable_phone.number }}
Screenshot: Salesforce Update node mapping
5
Test the workflow
Click Test workflow in n8n, then create a new lead in Salesforce. The top flow will send the enrichment request. After a few seconds, the webhook will receive the results and update the lead. Check your Salesforce lead to confirm.
Screenshot: Workflow execution results
6
Activate the workflow
Toggle the workflow to Active. Make sure to use the production webhook URL (without webhook-test in it). The test URL only works during manual testing.
About Custom Fields: The leadId custom field is essential. When FullEnrich sends the results back via webhook, it only returns enrichment data by default. Custom Fields let you pass through the Salesforce Lead ID so you know which lead to update.
Not using Salesforce? You can swap the Salesforce nodes for any other CRM (HubSpot, Pipedrive, Zoho, etc.). The FullEnrich + Webhook pattern stays the same. Just pass the right record ID as a custom field.

Troubleshooting

ProblemSolution
Webhook never receives dataMake sure you're using the production webhook URL (without webhook-test). The test URL only works during manual testing. Also make sure the workflow is Active.
Lead ID missing in webhook dataCheck that you added leadId as a Custom Field in the FullEnrich node. It should appear in $json.body.datas[0].custom_fields.leadId.
FullEnrich returns no emailMake sure First Name, Last Name, and Company are mapped. Adding the company domain (Website field) improves match rates.
Salesforce update failsCheck that the Lead ID expression is correct and that your Salesforce credentials have edit permissions on the Lead object.
